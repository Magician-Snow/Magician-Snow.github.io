    多线程编程

    1. 多线程编程的本质就是并发编程

    ​ 并发编程的本质是指的发挥出所有硬件的最大性能

    ​ Java是为数不多的真正支持有多线程并发编程的开发语言，所以在整体的处理性能上是最高的

    ​ 而python是多进程编程的注意区分

    2. 进程

    ​ 传统硬件只有一个CPU(单核CPU)，为了发挥硬件的全部性能，引入了多进程编程模式

    • 进程指的是一个具有一定独立功能的程序关于某个数据集合的一次运行活动
    • 进程是系统进行资源分配和调度运行的基本单位
    • 进程实体中包含有三个组成部分：“程序”、“数据”、“PCB(进程控制块)”

    3. 单进程运行与多进程运行的区别

    ​ 单进程运行
    ​ 所有的系统资源都为一个进程服务，该进程资源独享

    ​ 多进程运行
    ​ 在没有拓展原始系统硬件资源的前提下，利用一些算法，实现多个进程的并行执行

    ​ 在每个进程执行时要独享全部资源，即在同一个时间点上只会有一个进程执行

    4. 线程

    ​ 线程是在进程基础上的进一步划分，可以达到更快的处理性能，任何一个进程的启动都是非常缓慢的，线程性能上远高于进程，但是线程不可以离开进程存活

    ​ 进程与线程图示

    ​ 每一个进程都只有一个自己的重要处理单元，如果要想实现CPU(变量信息等)的共享，则必须利用线程来描述

    ​ 一般来说，每一块CPU只会有一个线程执行，但有一些CPU可以使用超线程技术，设计出若干个多线程的执行状态，这叫超线程

    5. 并行编程

    ​ 随着硬件和软件技术的发展，硬件中的CPU出现了多核状态，理论上多核CPU的多进程执行称为并行编程

    6. 纤(协)程

    ​ 在进程和线程的概念之上还有一个叫纤(协)程，是在线程基础上进一步划分，并不是像进程和线程一样由硬件支持，协程用的是由程序控制的
    ​ Java并没有支持有多协程编程（不代表以后没有），像python是支持多协程的

    7. 多线程编程起步

    ​ 所有的Java程序是通过主方法完成的，主方法作为程序的起点。而要进行多线程的编程也需要一个线程的起点结构，此结构称为线程类，所有的线程类是有继承要求的，可以有三种实现模式：继承Thread类、实现Runnable接口、实现Callable接口(两个接口更重要)

    ​ 多线程的几种状态

    ​

    ​ 所有的系统的资源是有限的，不管多线程还是多进程都必须在执行一段时间之后让出资源，交由其他的线程继续执行

    1. 创建过程：开发者定义好了相应的线程类对象(每一个Thread类的实例)
    2. 就绪状态：多线程的启动依靠的是start()方法，当调用start()方法的时候所有的线程不是立即执行，而是将进入到等待状态，等待CPU进行调度
    3. 执行状态：当CPU执行调度到了某个线程对象之后，该线程对象开始执行run()方法或call()方法，此时不是持续占用资源，而是在一段时间之后(一共时间片时间)，该线程就需要暂停执行
    4. 阻塞状态：当某个线程不再执行时(中断、休眠或调度失败)，那么所有的线程将进入到阻塞状态，如果此时线程没有执行完毕，则由阻塞状态切换到就绪状态，重新等待CPU执行调度
    5. 终止状态：如果多线程的执行体执行完毕或者被强制性的结束，那么就将进入到终止状态，终止状态不会再进入到就绪状态，即该线程对象将不会被继续执行

    CPU与多进程

    1. 多进程是管理CPU提出来的，通过管理明白CPU而后会管理其他硬件，所以说多进程是操作系统的核心

    2. CPU的工作原理

    ​ 把一个程序存放在内存中，设置一个地址，CPU根据这个地址发出一条取址的命令，即把这个地址从地址总线上由内存发给CPU，CPU得到程序后开始解释执行

    ​ CPU的工作就是不断的自动取址执行

    3. CPU的管理

    ​ 设置好PC初值就行（PC是寄存器），把PC初值设置在一段程序的开始地址

    ​ 问题
    ​ 但是如果只是设置好PC初值就等待程序执行会出现一个问题，当程序开始磁盘读写(I/O)时(很慢)，CPU不工作进入等待，而一般一个程序计算时间远比磁盘读写时间少，则造成CPU利用率很低，甚至为0

    ​ 解决方式：多到程序、交替执行(让CPU更好的工作)
    ​ 当CPU执行程序执行不下去时，就切出换到另外的程序去执行。等再执行不下去再切出，由此使得CPU忙碌，但所有程序终会在来回切换中执行完

    ​ 一个CPU上交替的执行多个程序叫做并发

    4. 怎么让CPU并发的工作

    • 控制PC进行切换
    • 记录程序切出去时的程序执行到的地址，样子（运行的程序和静态程序不一样）

    5. 刻画运行程序的样子：进程

    ​ 一个运行中的程序就是一个进程

    6. 用进程描述CPU管理

    ​ 启动一个进程让CPU去执行这个进程，此时CPU工作；而让CPU更好的工作则是操作系统启动多个进程，让CPU去跑这多个进程，提升CPU利用率。

    (3).流的简介

    ​ 流指的是数据的处理方式，以及目标的内容的处理机制，所有的流都分为输入流或输出流，Java针对输入流和输出流分为了两种类型：

    • 字节操作流：OutputStream（字节输出流）、InputStream（字节输入流）
    • 字符操作流：Write（字符输出流）、Reader（字符输入流）

    ​ 字节与字符最大的区别体现在对中文的处理上
    ​ 注意：不管使用哪一种操作流，本质上的操作流程都是类似的

    ​ 所谓输入流与输出流是相对的，而Java会提供一组标准的操作形式

    (10).字节流和字符流的区别

    ​ 字节操作属于基础的二进制数据流的操作形式，在网络、文件、图片、音乐等操作之中几乎都是以字节数据为主的，但是字节数据不方便进行中文的处理，所以对于中文的处理应该使用字符流。

    ​ 在进行网络传输或者磁盘数据存储的时候，所有存放的内容都属于字节数据


    ​ 缓冲区：实现字符与字节的转换

    ​ 实际上如果想要观察出字符流的缓冲区的存在，只需要在输出的时候取消close()方法的调用即可，因为此方法一旦调用实际上会自动清空缓冲区的内容，随后才会将数据写入到目标终端设备之中，如果此时由于某些原因就是无法进行字符输出流的关闭，则可以使用由Flushable接口继承而来的flush()方法进行手工的强制清除。

    ​ 本质上传递的所有内容只有字节，而字符是程序对字节的包装，通过代码可以发现字节流和字符流的使用非常相似，一般以字节流方式为主，只有进行中文操作时才会考虑字符流处理

    (11).字节流和字符流的转换（转换流）

    ​ 为了方便实现字节流和字符流的转换处理，提供了两个类：OutputStreamWriter和InputStreamReader

    1. OutputStreamWriter

      在构造方法中，需要接收一个OutputStream的对象，而后再向上转型变为字符流

    2. InputStreamReader

      通过继承的关系，InputStreamReader、OutputStreamWriter都是字符流的子类，同时又可以通过构造方法接收InputStream类和OutputStream类的实例，再向上转型为字符流

      转换流在一些系统类库的操作之中比较常见，有些情况系统只会给用户字节流，但是为了方便处理需要进行字符流转换，这个时候才会使用到这个流

      注意：这种转换在类的定义时也有体现，如：OutputStreamWriter是FileWriter的父类，InputStreamReader是FileReader的父类，即转换流是字符输入输出流的父类

    3. 操作系统的表层

    ​ 表层是通过应用使用操作系统的接口
    ​ 操作系统中的接口就是一个让用户操作电脑的一个界面，比如windows，它有三个接口：

    1. cmd：windows上按win+r打开运行，输入cmd就会打开命令提示符，这个是系统管理员接口。
    2. explorer.exe:你打开任务管理器会看到explorer.exe，这个就是windows的桌面接口。
    3. api接口，这个是面向程序员的接口

    ​ 通过IP(指针)把程序载入到内存(存储器)中，再载入到CPU(运算器、控制器)中解释执行，再输出

    ​ 计算机工作的本质：取指执行

    5. 计算机的开机(x86 PC)

    1. x86 PC刚开机时CPU处于实模式（实模式的寻址CS:IP(CS左移4位+IP)）
    2. 开机时，CS=0xFFFF；IP=0x0000
    3. 寻址0xFFFF0（ROM BLOS(基本输入输出)映射区(此处此时为固化的)）
    4. 检查RAM(内存)，键盘，显示器，软硬键盘（执行过不去就是硬件坏了）
    5. 将磁盘0磁道0扇区(操作系统的引导扇区)读入0x7c00处（一个扇区为512字节）
    6. 设置CS=0x07c0，IP=0x0000（BLOS跳到这个自己写的引导扇区执行，引导扇区为启动设备的第一个扇区）

    ​ 这512个字节为开机时发生的事情
